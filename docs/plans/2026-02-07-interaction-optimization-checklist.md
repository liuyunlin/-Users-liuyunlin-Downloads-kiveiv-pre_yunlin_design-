# 交互优化清单（第二轮/第三轮收口）

## 一、已确认问题
- 问答页不应提供“筛选/批量”能力，用户目标是快速找会话并继续提问。
- 问答二级侧栏信息层级不清晰，主操作与管理操作混杂。
- 知识库页把“筛选”和“批量”合并成一个入口，违背任务心智模型。
- 知识库管理中“进入批量模式”和“设置筛选条件”路径耦合，造成认知跳转。

## 二、本轮已修复
- 问答页：移除“筛选/批量”入口与批量删除流程，仅保留单会话操作（重命名/置顶/删除）。
- 问答页：重构二级侧栏为 `New chat -> Search chats -> 置顶 -> 最近对话` 的顺序。
- 问答页：收起态只保留最小核心动作（新建、展开）。
- 知识库页：将“筛选与批量”拆为两个按钮：`筛选`、`批量管理`。
- 知识库页：筛选弹窗仅处理筛选；批量弹窗仅处理勾选与删除，不再混用。

## 三、交互规范（本次收口）
- 列表类页面的二级操作使用“二级弹窗”承载，不在主列表区域堆叠复杂控件。
- `筛选` 与 `批量` 必须分入口，禁止再出现“筛选与批量”合并按钮。
- 问答页面不纳入批量操作范畴，只允许单条会话管理。

## 四、待你验收
- 问答页侧栏视觉是否与参考图一致（层级与可见性）。
- 知识库筛选与批量拆分后，是否符合演示路径。
- 删除确认文案与风险提示是否需要再强化。

## 五、本轮新增（自动流转）
- 去除知识库工作区顶部“下一步”手动按钮，改为状态驱动自动流转。
- 上传完成后，在解析页新增明确状态提示与双按钮：`继续上传`、`开始解析`。
- 解析完成后自动触发切分；切分完成后自动触发向量化；向量化完成后自动触发索引构建。
- 自动流转时采用页面内轻提示（非打断）告知当前阶段变化。
- 切分阶段移除进度弹窗，改为列表内每文档环形进度显示。
- 上传模块修复为支持多文件批量入队（不再只处理第一个文件）。
